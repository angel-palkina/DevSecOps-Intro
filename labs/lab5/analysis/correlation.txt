=== Key Insights ===

SAST (Static Analysis) Strengths:
   Finds code-level vulnerabilities before deployment
   Detects: hardcoded secrets, SQL injection patterns, insecure crypto, XSS in templates
   Fast feedback in development phase (3-5 minutes)
   Pinpoints exact file and line number
   Zero runtime overhead

DAST (Dynamic Analysis) Strengths:
   Finds runtime configuration and deployment issues
   Detects: missing security headers, auth flaws, server misconfigs
   Authenticated scanning reveals 10x more attack surface
   Proves exploitability (SQLmap extracted real data)
   Tests actual deployed environment

Complementary Coverage:
  - SAST found SQL injection CODE patterns (6 instances)
  - DAST confirmed SQL injection is EXPLOITABLE in runtime
  - SAST found hardcoded JWT secret in source
  - DAST found missing auth headers allowing token theft

Recommendation: Use BOTH approaches for comprehensive security coverage
  - SAST in CI/CD (pre-commit, PR checks)
  - DAST in staging/QA (pre-deployment validation)
